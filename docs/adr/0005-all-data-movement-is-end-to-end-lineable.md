# All data movement is end-to-end lineable through the catalog

Every read and every write in the pipeline goes through a catalogued object — a table, view, materialised view, or volume registered in the catalog. Raw-file writes that bypass the catalog (`df.write` to ABFSS paths, DBFS shortcuts, mounted blob paths) are forbidden, because they break the lineage chain that downstream consumers, auditors, and impact-analysis depend on. This follows the KRM "Lineage over de hele keten" principle. On Databricks specifically this means every table, view and volume is registered in Unity Catalog and the lineage graph is therefore complete by construction; on another platform the same principle would require explicit lineage annotations, but the rule itself is platform-independent.
