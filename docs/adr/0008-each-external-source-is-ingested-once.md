# Each external source is ingested once into the lakehouse

Every external data source — a parquet folder, an operational database, a SaaS API — is ingested into the lakehouse exactly once per change event. All downstream layers read from the lakehouse copy, never back from the original source. This follows the KRM "Eénmalige ontsluiting van gegevens" principle and gives three properties at once: lineage stays complete (every downstream object can be traced back to its lakehouse origin), the source system is insulated from analytical query load, and the integration and datamart layers can re-process from history without needing the source to be re-pulled. The cost is one extra copy of the data at rest in the lakehouse; on a Delta-backed catalog with cheap object storage this trade is one-sided.
