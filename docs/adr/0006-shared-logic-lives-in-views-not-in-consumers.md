# Shared transformation logic lives in views, not in consumers

When the same business rule, type-fix, or derivation is needed by more than one consumer — a dashboard widget, a downstream pipeline, an AI/BI Genie space, an ad-hoc analyst query — that rule lives in exactly one view or materialised view that all consumers read from. Consumers do not re-implement the rule. This follows the KRM "Hergebruik logica in alle afnemende stromen" principle and is the reason the historical layer always exposes a view on top of the storage table (so window-function derivations like validity periods and previous-version keys are shared by definition), and the reason dimensions are themselves views rather than copies. The cost — slightly more compute on each read of a non-materialised view — is accepted in exchange for a single canonical implementation of every shared rule.
